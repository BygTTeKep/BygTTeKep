#nodejs 
```js
const mem = process.memoryUsage();
console.log(mem)

{
  rss: 73400320,        // ~70 MB → общая память процесса
  heapTotal: 23461888,  // ~22 MB → выделенная куча
  heapUsed: 12345678,   // ~12 MB → реально используемая куча
  external: 4320000,    // ~4 MB → нативные буферы
  arrayBuffers: 120000  // ~0.1 MB → ArrayBuffer
}
```

`heapTotal` - Общий размер выделенной кучи V8 (JS-объекты) (**входит в RSS**)
- Показывает размер выделенной кучи (может расти при GC).

`heapUsed` - Используемая часть кучи (реально занято JS-объектами) (**входит в RSS**)
- Показывает, сколько JS-памяти реально используется (утечки в коде).

`external` - Используемая часть кучи (например, `Buffer`, `C++` ресурсы) (**входит в RSS**)
- Помогает искать утечки через `Buffer` или нативные зависимости.

`arrayBuffer` - Объём, занятый `ArrayBuffer` и `SharedArrayBuffer` (**входит в RSS**)
`stack` - Память под стек вызовов и контекст выполнения (**входит в RSS**)

`rss` - все, что физически зарезервировано в оперативке процессов NodeJS (не обязательно всё используется прямо сейчас JS-объектами).
- Показывает общий след процесса в RAM (на уровне ОС).
