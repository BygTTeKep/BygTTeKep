Миграции - это система контроля версий для базы данных, которая позволяет управлять изменениями схемы бд безопасным и организованным образом.
# Зачем нужны миграции
1) Версионность - отслеживание истории изменений БД.
2) Совместная работа - несколько разработчиков могут работать с одной бд
3) Воспроизводимость - одинаковое состояние бд на всех окружениях.
4) Откат измеенний - возможность вернуться к предыдущей версии схемы
## Лучшие практики

1. **Никогда не используйте `synchronize: true` в продакшене**

2. **Каждая миграция должна быть атомарной**

3. **Всегда пишите методы `down` для отката**

4. **Тестируйте миграции на staging окружении**

5. **Храните миграции в системе контроля версий**

6. **Не изменяйте уже примененные миграции**
## Когда использовать `migration:create`?
(создает пустую миграцию где руками прописывать все надо)

1. **Сложные миграции** с несколькими шагами
2. **Миграции с данными** (заполнение, преобразование)
3. **Оптимизация БД** (индексы, партиции)
4. **Специфичные SQL-запросы**, которые TypeORM не умеет генерировать
5. **Транзакционные операции**
