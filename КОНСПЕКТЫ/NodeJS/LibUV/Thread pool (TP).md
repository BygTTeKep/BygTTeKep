TP - дополнительный пул потоков, который используется не для исполнений JS, а для выноса блокирующих или CPU/OS - bound операций за пределы event loop

# Что именно выполняет TP
1) работу с файлами
2) криптография
3) compresison/decoopmression
4) dns частично
По умолчанию в TP 4 потока

Упрощенная работа
1) js вызывает например fs.readFile
2) libuv кладет задачу в queue TP
3) свободный worker поток обрабатывает 
4) После завершения резульатт кладется в event queue
5) Event loop вызывает js callback
