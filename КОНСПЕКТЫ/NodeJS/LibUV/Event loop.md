Event loop - Это механизм реализованный в libuv, который используется в Nodejs для выполнения асинхронного js кода. Концептуально это цикл который проходится по всем фазам в строгом порядке, каждая из которых выполняет свою очередь колбэков.

Почему задачи выполняются в строгом порядке?  Если бы задачи выполнялись в произвольном порядке, мы бы не могли гарантировать результат выполнения одной и тойже задачи одинаковым. Тоесть порядок существует для избежания race condition на уровне js кода и соблюдения детерменированости

Почему Event loop не завершается когда мы запускаем сервер?
Event loop сам по себе не является бесконечным, он может завершиться  если
- нет активных handles
- нет активных request
- нет запланированных timers
- нет ожидающего I/O
При запуске сервера
- создается TCP-сокет
- Он регесрируется в libuv
- добавляется в active handle
- libuv начинает слушать события ОС
Тоесть есть активный handle и поэтому event loop не завершается
