Redis stream - это струкутура данных в redis, предназначенная для работы с потоками сообщений. По сути это похоже на журнал событий, где каждый элемент имеет уникальный идентификатор (ID) и может содержать несколько полей (key-value)

Основные идеи
- Упорядоченный поток сообщений. Сообщения хранятся в порядке поступления
- Каждое сообщение имеет ID - автоматически генерируется при добвалении и обычно выглядит как `<timestamp>-<sequence_number>`.
- Можно читать сообщения разными клиентами, включая группы потребителей.
