Redis Streams поддерживает **группы потребителей**, чтобы несколько клиентов могли читать один и тот же поток без пересечения сообщений.
```
XGROUP CREATE mystream mygroup $ MKSTREAM

```

- `mygroup` — имя группы.
- `$` — начинать с новых сообщений.
Чтение сообщений в группе:
`XREADGROUP GROUP mygroup Alice STREAMS mystream >`
- `Alice` — имя потребителя в группе.
- `>` — получить только новые, ещё не прочитанные сообщения.
Подтверждение прочтения:
```
XACK mystream mygroup 1687556234240-0
```
