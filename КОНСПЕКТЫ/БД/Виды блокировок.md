# Пессисимтические блокировки
Они подразумевают что конфликт при параллельном доступе произойдет и поэтому сразу накладываются блокировки на строки/таблицы. Это гарантирует что данные будут изменяться только одним процессом или транзакцией, предотвращая возможные ошибки.

## Row level locks
### For update
Блокирует строку для изменения в рамках текущей транзакции. Другие транзакции не могут изменять эти строки до завершения текущей и встают в очередь(но могут читать)

### For share
Позволяет читать строки другим транзакциям, но блокирует возможность их изменения

### For no key update
Блокирует строки для изменений, но позволяет другим транзакциям делать только чтение или блокировать строки для обновлений.

## Table level locks

### Access share lock
используется для операций чтения данных. Самый легкий тип

### Row Exclusive lock
Накладывается при изменении данных в таблице(вставка, обновление)

### Exclusive lock
Накладывается при операции, которая изменяется структуру таблицы.


## SKIP LOCKED
позволяет при выборке данных пропускать заблокированные строки
