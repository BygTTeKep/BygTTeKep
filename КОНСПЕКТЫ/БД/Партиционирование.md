Партиционирование - это процесс разбиение большой таблицы или индекса на более мелкие части, называемыми партициями, с целью улучшения производительности, управляемости и масштабируемости. Каждая партиция может хранится отдельно и обрабатываться независимо

# Зачем нужно
1) повышает производительность запросов
	1) Система может сканировать только нужные партиции а не всю таблицу
2) Упрощенное управление данными
	1) легко архивирвоаь или удалять данные, просто удаляя нужную партицию
	2) Можно перемещать партиции между дисками или серверами
3) Балансировка нагрузки и масштабирование

# Основные виды партиций
1) Range(диапазон)
	1) Строки попадают в партиции на основе диапазона значений
2) List(список)
	1) каждая партиция содержит опеределенные значения
3) Hash
	1) Строки распределяются между партициями на основе хэш функций
# Примеры в pg
```sql
CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    order_date DATE,
    amount NUMERIC
) PARTITION BY RANGE (order_date);

CREATE TABLE orders_2023 PARTITION OF orders
    FOR VALUES FROM ('2023-01-01') TO ('2023-06-01');

CREATE TABLE orders_2024 PARTITION OF orders
    FOR VALUES FROM ('2024-01-01') TO ('2024-06-01');
```

# Примеры MySQL
```sql
CREATE TABLE users (
    id INT,
    name VARCHAR(50)
)
PARTITION BY HASH(id)
PARTITIONS 4;

```
# Важные моменты
- индексы и  внешние ключи могу тработать иначе 
- Не все типы партиционирования поддерживаются во всех СУБД
- Нужно продумать ключ партиционирования заранее, чтобы запросы могли эффективно использовать партиции.
