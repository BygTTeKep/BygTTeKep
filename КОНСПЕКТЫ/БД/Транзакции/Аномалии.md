# Dirty read
Транзакция читает данные, которые другая транзакция еще не закомитила
## Пример
```
T1 UPDATE users SET balance = balance - 100 WHERE id = 1;
T2 SELECT balance FROM users WHERE id = 1;
T1 ROLLBACK;
```

# Non-repetable read
Одна транзакция читает одну и туже строку и получает разные значения
## Пример
```
T1 SELECT balance FROM users WHERE id = 1;
T2 UPDATE users SET balance = 200 WHERE id = 1;
T1 SELECT balance FROM users WHERE id = 1;
```
## Как исправить
повышаем уровень изоляции транзакции до repetable read или serializable или используем SELECT ... FOR UPDATE

# Phantom Read
При повторном выполнении запроса возращается разное количество строк
## Пример
```
T1 SELECT * FROM orders WHERE price > 100;
T2 INSERT INTO orders(price) VALUE (150);
T1 SELECT * FROM orders WHERE price > 100;
```
## Как исправить
Повышаем уровень изоляции до serializable или используем predicate lock(pg), gap locks(mysql)

# Lost Update
Две транзакции читают одно значение и записывают его, в итоге одна перетирает другую

## Пример
```
T1 SELECT balance = 100;
T2 SELECT balance = 100;

T1 UPDATE balance = 150;
T2 UPDATE balance = 130;
```
## Как исправить
Повышаем уровень изоляции тразнакции до repetable read/serializable или используем SELECT ... FOR UPDATE

# Write Skew
Есть какоето внешнее условие. Две транзакции читают одни и теже данные, но обновляют разные строки.

## Пример
условие: хотябы 1 врач есть на смене
```
T1 проверяет, получает 2 врача на есть
T2 проверяет, получает 2 врача на есть

T1 делает doctor1 unavailable
T2 делает doctor2 unavailable
```
## Как исправить
Повышаем уровень изоляции транзакции до serializable / используем SELECT ... FOR UPDATE
