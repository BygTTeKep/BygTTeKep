InnoDB - это основной (и по умолчанию) движок хранения данных в MySQL
тоесть это модулб который отвечает за:
- физическое хранение таблиц и индексов
- транзакции
- блокировки
- контроли целостности данных
- MVCC
`
Можно сказать просто:

> MySQL — это оболочка (SQL-сервер),  
> а InnoDB — "мотор под капотом", который хранит и управляет данными.

# Ключевые особености InnoDB
- ACID 
- MVCC
- Foreign Keys
- Row-level locking
- Crash recovery
- Буферный пул
- Full-text и spatial индексы
## Где хранятся данные InnoDB

InnoDB хранит свои данные в:
- таблиспейсах (`.ibd` файлы или общий `ibdata1`),
- redo log файлах (`ib_logfile0`, `ib_logfile1`),
- undo log (в отдельном пространстве или внутри общего).
```
/var/lib/mysql/
 ├── ibdata1             ← системный tablespace
 ├── ib_logfile0/1       ← redo logs (журнал изменений)
 ├── mydb/
 │    ├── users.ibd      ← данные таблицы users
 │    └── users.frm      ← структура таблицы (в старых версиях)

```
## Буферный пул (InnoDB Buffer Pool)

Это сердце InnoDB 
-  кэш в памяти, где хранятся страницы таблиц и индексов.
`innodb_buffer_pool_size = 4G
> (рекомендуется до 70–80% оперативной памяти сервера).



Исторически были и другие движки:
- **MyISAM** — старый, без транзакций;
- **MEMORY** — хранит данные в RAM;
- **ARCHIVE**, **CSV**, **NDB** — для особых случаев.
Но с 5.5 и выше:

> InnoDB — движок **по умолчанию** и **рекомендуемый во всех случаях**.
