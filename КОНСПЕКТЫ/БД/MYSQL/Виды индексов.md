# BTREE
Самы распространненый используется по умолчанию. Эффективен для поиска по равентсву, диапазону и сортировки

# hash
Используется в таблицах типа `MEMORY` очень быстрый по точным совпадениям, но не подходит для дипазаонов

# FULLTEXT
Специальный индекс для полнотекстового поиска по текстовым данным, используется c `MATCH() .. AGAINST()`

# SPATIAL
Для работы с пространныственными данными(геометрия, точки), использует R- деревья, требует движака хранения, поддерживающего пространственные индексы
