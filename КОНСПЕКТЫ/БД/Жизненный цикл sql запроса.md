Жизненный цикл sql запроса начинается с парсинга, тоетсь:
- СУБД проверяет синтаксис запроса
- Преобразует SQL-код в машиночитаемое представление
затем  Планировщик и оптимизатор
- Парсер передает запрос планровщику
- Оптимизатор использует статистику базы данных(кол-во строк, индексы), чтобы найти саммый эффективный план выполнения запроса из всех возможных
- Создается план запроса(Query plan) - последовательность операций
затем этот план выполняется, задействуя кэш, память или диск и возращает результат 
При этом запрос выполняется в определенном порядке
1) FROM/JOIN
2) WHERE
3) GROUP BY/HAVING
4)  ORDER BY
5) SELECT
