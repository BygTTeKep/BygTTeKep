Bounded Context - Это граница, нутри которой термины, правила и ммоделли имеют однозначный смысл

# Зачем делить домен
DDD не делит по таблицам, сервисам или микросервисам
он делит по:
- ответственности
- правилам
- языку бизнеса
- скорости изменений

# Признаки что BC нужен
- Если одно слово значит разное
- Много условной логики по ролям
- Разные команды "владеют разными частями"

# Как BC общаются между собой
- события
- ACL
- read-модели
- контракты
Context Map - обязательный инструмент
CM Отвечает:
- какие контексты есть
- кто от кого зависит
- типы отношений
Типы связей:
- Upstream/Downstream
- ACL
- Conformist
- Shared Kernel
