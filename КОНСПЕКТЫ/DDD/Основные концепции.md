# Domain
Предметная область бизнеса:
- заказы, платежи, пользователи, статусы, правила
- то, ради чего система существует
DDD не про инфраструктуру, а про бизнес смысл

# Ubiquitos Language (UL)
Общий язык между:
- разработчиками
- бизнесом
- аналитиками
Этот язык:
- используется в коде
- использется в документации
- совпадает с терминами бизнеса

# Bounded Context
Границы, внутри которых термины имеют смысл

# Entity 
ОБъект с идентичностью, которая важнее состояния
`Order (id=123)`
Даже если все поля изменились, это все еще тот же Order

# Value Object (VO)
Обхект без идентичности, опеределяется только значением
Примеры:
- Money
- Email
- Date Range
Характеристики:
- immutable
- сравнение по значению
- не имеет id

# Aggregate / Aggreate Root
Aggregate  - кластер сущностей, управляемых как единое целое.
Aggragete Root - единственная точка доступа.
```
Order (ROOT)
|
|- OrderItem
|- DeliveryInfo
```
Правила:
- извне можно работать только через root
- инварианты проверяются только внутри аггрегата
- одна транзакция - один аггрегат

# Repository
Абстракция доступа к аггрегатам.
Важно:
 - репозиторий работает с доменными объектами  
 - Не возвращает DTO, rows, JSON

# Domain Service
Логика, которая:
- не принадлежит одной сущности
- но является бизнес правилом
Пример:
- расчет комиссии
- распределение заказов между курьерами

# Domain Events
Факт который уже произошел в домене
Примеры:
- OrderCreated
- PatmentConfirmed
Используется для:
- Слабой связности
- интеграции между bounded contexts
- eventual consistency
