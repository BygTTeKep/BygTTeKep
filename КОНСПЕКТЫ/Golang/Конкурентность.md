Конкурентность - это спрособ организовать программу так, будто несколько задач выполняются одновременно

Свойства
- задачи перекрываюстя во времени
- не требуется несколько CPU ядер
- это дизайн, а не характеристика выполнения
Задачи не выполняются одновременно, но ни одна задача не выполняется бесконечно

A, B, C - конкуренты если они существуют одновременно и порядок доступа к ОБЩИМ ресурсам не фиксирован заранее. 

В один момент времени может сущестовать несколько задач в одном потоке, но в стеке потока будет выполняться только стек одной задачи(это про то что 1 поток = несколько задач)
