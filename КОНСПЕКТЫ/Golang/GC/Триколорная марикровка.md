Триколорная маркировка - это алгоритм, который используется gc для определения какие объекты в куче еще используются, а какие можно удалить
## зачем нужно
Когда gc запускается, он должен решить какие объекты достижимы(reachable) а какие не достижимы(garbage). Но возникает проблема, программа продолжает работу параллельно с работой gc и gc может удалить объекты которые еще используются

## Основная идея
Все объекты в куче делятся на 3 цвета
- Белый - объект еще не проверен Gc
- серый - объект найден, но его ссылки еще не были просмотренны
- черный - объект полностью проверен
Главное правило: черный объект не можетт ссылаться на белый объект
## ка проходит gc
1) все объекты считаются белыми
2) gc берет корне (roots)(stack переменные, глобальные переменные, регистры cpu) и помечает их серыми
3) gc обрабатывает все сервые объекты: берет серый объект просматривает все его поля, найденные объекты помечает серым,  а сам становится черным, это происходит пока все объекты не пометятся серым цветом
4) После маркировки: черные живые, белые мусор
В go gc выполняется конкурентно и может возникнуть ситуация когда Gc удалил живой объект, для решения это проблемы используется write barrier.
Когда программа делает запись
```
obj.field(A) = ptr(B)
A(black) -> B(white)
```
write barrier делает B сразу серым объектом, и gc не потеряет его
В Go используется гибридный write barrier он комбинирует
1) Dijkstra barrier
2) Yuasa barrier
