# Проблема которую решает
Есть общий алгоритм, но некоторые шаги должны отличаться
# Симптомы
- несколько сервисов с почти одинаковым кодом
- отличие в 1-3 шагах
- дублирование логики
- изменения нужно делать в нескольких местах
# Суть
Template method определяет скелет алогоритма, а отдельные шаги делегируются подклассам
# Ключевая идея
- алгоритм фиксирован
- вариативны части, а не весь процесс
# Что именно фиксируется
- порядок шагов
- инварианты алгоритма
- защиту от неправильного использования
# Когда оправдан
- алгоритм один
- шаги частично отличаются
- порядок критичен
- бизнес процесс стабилен
# Когда антипаттерн
- вариация много
- алгоритм часто меняется
- шаги условные и динамические
