# Какую проблему решает
COR  решает проблему жесткой привязки отправителя запроса к конкретному обработчику
Chain позволяет динамически выстроить конвейер обработки, не меняя клиента 

---
# Симтомы
- последовательные проверки / шаги обработки
- много if/else if
- разные сценарии требуют разного набора шагов 

---
# Суть

Суть Chain в том, что:
- запрос проходит через цепочку обработчиков
- каждый обработчик
	- либо обрабатывает запрос
	- либо передает его дальше
- отправитель
	- не знает, кто именно обработал запрос
	- не знает, сколько шагов будет выполнено

---
# Ключевая идея
Разделить сложную обработку на независимые шаги, связанные только договором
"Если я не обработал, передай дальше" 

---
# Когда оправдан
- есть
	- валидация
	- фильрация
	- middleware
	- policy check
- порядок шагов может меняться
- шаги
	- независимы
	- легко тестируются отдельно


---
# Когда антипаттерн
- каждый шаг знает про следующий
- есть жесткая бизнес логика порядка
- запрос обязательно должен пройти все шаги
- цепочка становится слишком длинной
