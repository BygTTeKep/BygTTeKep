# Какую проблему решает
UoW решает проблему несогласованости данных при выполнении одного бизнес сценария
UoW обеспечивает
- атомарность
- констстентность
- единую фиксацию измененний

---
# Симтомы
- один бизнес сценарий 
	- меняет несколько аггрегатов
	- используется несколько репозиториев
- транзакции размазаны по сервисам дублируются
- сложно гарантировать rollback


---
# Суть
- отслеживает изменненые объекты
- фиксирует их одним коммитом
- управляет
	- транзакцией
	- жизненным циклом persistence
- служит границей бизнес-операциии


---
# Ключевая идея
Сгрупировать изменения и применить их как единое целое
Клиент
- не думает о транзакциях
- не знает сколько репозиториев участвует

---
# Когда оправдан
- есть слоэные use case
- требуется ACID, rollback
- используется DDD, Repository
- есть явные границы транзакций


---
# Когда антипаттерн
- используется для архитектурной чистоты
- бизнес сценарии простые
- транзакции маленькие, независимые
- используется поверх event riven flow
