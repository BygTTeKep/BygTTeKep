# Проблема которую решает
Поведение системы зависит от условий которые современем будут меняться
# Симптомы
- длинные if/else
- switch по типу, статусу, флагу
- бизнес логика размазана по сервису
- каждое новое правило ломает старое
# Суть
Strategy инкапсулирует взаимозаменяемые алгоритмы и позволяет выбирать их во время выполнения
# Ключевая идея
- поведение выносится в отдельные компоненты
- клиент знает интерфейс, а не реализацию
# Когда оправдан
- есть 2+ варианта поведения 
- ожидается рост количества вариантов
- поведение бизнес критично
- условия выбора стабильны
# Когда антипаттерн
- когда вариантов мало и они стабильны 
- условие не бизнесовое, а техническое 
- логика простая и не растет
