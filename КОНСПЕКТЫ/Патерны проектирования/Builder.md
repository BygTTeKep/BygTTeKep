# Проблема которую решает Builder
Объект сложный, имеет много опциональных параметров и должен быть валидным
# Симптомы
- конструктор с 10+ параметрами
- null/undefined
- порядок аргументов важен
# Суть
Builder пошагово собирает объект, контролируя корректность состояния

# Ключевая идея
- разделение процесса построения и результата
# Когда оправдан
- сложные конфигурации
- неизменяемые объекты
- пошаговая сборка
- читаемость важнее краткости
# Когда антипаттерн
- объект простой
- мало параметров
- нет инвариантов
